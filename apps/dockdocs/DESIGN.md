# DockDocs 设计规范（统一标尺）

> 工作台 / 官网 / PWA / 桌面版**共用这一把尺**。新增任何 UI 前先读本文，做完对照 §六 验收。
> 目的：避免每页各自为政、每次新增重说一遍；未来所有形态保持一致。

---

## 一、对标 + 心法

**对标对象：Resend + Linear**（顶级 SaaS app-shell）。做任何组件先问一句："Resend / Linear 会怎么做？"

**五条心法：**
1. **简单 = 高级** —— 克制是高级，炫技是廉价。默认问"这个能不能删 / 更轻？"
2. **留白是设计** —— 留白不是空，是呼吸。内容聚焦一块 + 周围留白 > 铺满整个区域。
3. **安静** —— UI 不抢注意力，把注意力留给内容（导航、装饰越安静越高级）。
4. **浅色 = 深色同一套语言换配色** —— 浅色不是深色的淡版；同结构、同层次，只翻配色，同等精致。
5. **一致 = 高级的地基** —— 乱（字号乱、间距乱、颜色乱）= 廉价。一律走 token + 阶梯。

---

## 二、颜色 / 明暗（token 单一真源：`globals.css`）

**明度层级**（深色、浅色都必须清晰可辨）：
`背景 → 卡片(surface) → 边框(line) → 次要文字(muted/faint) → 主文字(foreground)`；accent 只点睛。

- **深色**：warm near-black 背景 + 略亮卡片 + 高对比近白文字 + 绿 accent 点睛
- **浅色**：白/极浅背景 + **浅灰卡片**（subtle 但分得清，别和背景糊在一起）+ 深字高对比 + 绿按钮点睛
- **accent（绿 `#3ecf8e`）**：只用于 CTA / 链接 / 选中 / 小点缀，**绝不滥用**；绿按钮用深字（global rule）
- 深度靠 `1px --line` 边框，**不靠阴影**

---

## 三、框 / 容器（响应式）

- **宽度**：填充可用区 + **max-width 上限**（操作框 `max-w-3xl~4xl`）+ `mx-auto` 居中 + 左右 padding
- **高度**：内容自然高度 + `min-h` 适中，**绝不 `flex-1` 撑满全区**，下方留白
- **内容**：框内水平垂直居中
- **4 状态**（有/无侧栏 × 最小/最大宽）：框永远「居中 + max-w 上限 + 高度适中 + 内容居中」，留白随屏幕变多、内容那块始终聚焦

---

## 四、间距 / 字号 / 字重

- **间距阶**：`4 / 8 / 12 / 16 / 24 / 32`（px）——只用阶，不随手填数
- **字号阶**：`11 / 12.5 / 13 / 14 / 16 / 18 / 22`（px）
- **字重**：标题 `400–500`（层级靠**字号**不靠加粗）；正文 `400`；mono 用于 eyebrow 小标签

---

## 五、组件标准

- **选中态**：`subtle bg(accent ~9%)` + 文字转 accent + 左 `2px` 细条 —— **不用大色块**
- **图标**：统一 `16px`、`stroke 1.5`、`currentColor`
- **按钮**：主操作 accent（深字）；次要 ghost/outline；**一屏一个主按钮**点睛
- **空状态**：一等公民 —— 居中卡片（图标 + 一句说明 + 一个明确动作），别留白板
- **上传框**：按 §三（Resend 式居中适中，**非铺满**）
- **hover**：subtle bg，平滑不突变

---

## 六、工具页信息架构（工作台空状态）

**工作台工具页 ≠ 官网工具页。** 官网面向 SEO 来的未决策陌生人（任务＝说服：benefits / steps / FAQ / social proof）；工作台面向已进来干活的人（任务＝帮他快速干完）。所以工作台工具页**只传 3 层信息，不堆营销**：

1. **价值** — 工具名（h1）+ **一句价值副标题**（这工具做什么 + 关键价值，一句话替代官网整块 benefits）
2. **操作** — 上传区（按 §三 框规格，居中适中）
3. **信任** — 隐私 / 可溯源一行（本地处理不上传 / 可溯源 badge），护城河，工作台也要露出

**绝不放**：benefits 三卡 / steps 教程 / FAQ / social proof —— 那是官网说服未决策者用的。

**布局**：整体**垂直居中** + 限宽（内容块 max-w ~560），留白均匀＝呼吸。（"太空"的真因不是信息少，是内容偏上、下方留白不平衡 → 垂直居中解决。）

**渐进式信息密度**（按需出现，不一刀切）：
- 新用户 / 空账户：极简（标题 + 副标题 + 上传 + 信任）
- 用过该工具后：底部出现**最近文件**（快速复用），不喧宾夺主
- 处理完（结果态）：给"下一步"建议（如转完 Word → 编辑 / 合并），把单工具变工作流、提升留存

## 七、验收 checklist（每个组件 / 页面过一遍）

- [ ] **对标**：到 Resend / Linear 那个级别了吗？
- [ ] **工具页**：3 层信息（价值副标题 + 操作 + 信任）齐、垂直居中、无官网营销块？
- [ ] **框**：居中 + max-w 上限 + 高度适中（没有 `flex-1` 铺满）？
- [ ] **明暗**：浅色、深色**都**层次清晰 + 对比足够？
- [ ] **一致**：字号 / 间距 / 颜色都走了阶 / token，没有随手值？
- [ ] **克制**：能删的删了？accent 没滥用？留白够？
- [ ] **状态**：选中 / hover / 空状态 / loading 都设计了？
- [ ] **零跳出**（工作台内）：点了之后人还在工作台里？
