# Dock One-Click Storage Maintenance

`scripts/storage-maintenance-one-click.mjs` provides a reusable storage maintenance entrypoint for Dock worktrees under `C:\Users\47203\Documents`.

## Dry Run

Dry-run is the default and deletes nothing:

```powershell
node scripts/storage-maintenance-one-click.mjs --root "C:\Users\47203\Documents" --mode dry-run
```

It writes:

- `C:\Users\47203\Documents\Dock-storage-maintenance-result.json`
- `C:\Users\47203\Documents\Dock-storage-maintenance-result.md`

## Apply Artifacts

This mode deletes only safe reproducible build and test artifacts:

```powershell
node scripts/storage-maintenance-one-click.mjs --root "C:\Users\47203\Documents" --mode apply-artifacts
```

Allowed artifact targets include `.next`, `out`, `test-results`, `playwright-report`, `.turbo`, `.cache`, and `apps/dockdocs/tsconfig.tsbuildinfo`.

## Apply Approved node_modules

This mode reads the approved node_modules cleanup plan and deletes only `approved=true` candidates:

```powershell
node scripts/storage-maintenance-one-click.mjs --root "C:\Users\47203\Documents" --mode apply-approved-node-modules
```

It does not generate new approvals.

## Full Approved Maintenance

This mode runs health monitoring, artifact cleanup, node_modules audit, cleanup plan generation, auto approval proposal, approved plan conversion, and approved-only node_modules apply:

```powershell
node scripts/storage-maintenance-one-click.mjs --root "C:\Users\47203\Documents" --mode full-approved-maintenance
```

It still skips the original Dock repository, dirty worktrees, conflict worktrees, and active branch worktrees.

## Build Validation

Build validation is optional:

```powershell
node scripts/storage-maintenance-one-click.mjs --root "C:\Users\47203\Documents" --mode dry-run --run-build-validation
```

Use `--skip-build-validation` to keep maintenance fast.

## Safety Boundary

The script does not run `git clean`, `git reset`, `npm cache clean`, merge, push, deploy, or PMO Board edits. It uses Node filesystem APIs for approved deletion work.

## Recommended Frequency

- Dry-run: after every large merge or deploy batch.
- Apply artifacts: after merge/deploy validation evidence is recorded.
- Apply approved node_modules: only after approval plans are reviewed.
- Full approved maintenance: only during a dedicated OPS storage window.

## Recovery

Build artifacts are regenerated by TypeScript, Next.js, and Playwright validation. Deleted `node_modules` directories can be restored with `npm install` in the affected worktree.
