# Dock Storage Cleanup Policy

This policy defines conservative cleanup rules for Dock worktrees under `C:\Users\47203\Documents`.

## Worktree Lifecycle

Use fresh worktrees for merge, deploy, and validation runs. Keep them only until the related merge report, deploy report, or QA evidence has been recorded. The original `Dock` repository is not a cleanup target.

## Safe Cleanup Targets

The safe cleanup script may remove only reproducible build and test artifacts:

- `apps/dockdocs/.next`
- `apps/dockdocs/out`
- `apps/dockdocs/test-results`
- `apps/dockdocs/playwright-report`
- `.turbo`
- `.cache`
- `apps/dockdocs/tsconfig.tsbuildinfo`

These files and directories are regenerated by install, build, TypeScript, or Playwright validation.

## Node Modules Policy

`node_modules` is excluded by default. It may be considered only when all of the following are true:

- The worktree is clean.
- The branch is confirmed merged or no longer active.
- The script is run with `--include-node-modules=true --apply`.
- The operator accepts that a later validation run will need `npm install` again.

## Deploy Worktree Retention

Deploy worktrees should be kept until production deploy ID, local validation, and post-deploy QA have been recorded. After that, build artifacts such as `.next`, `out`, and Playwright reports are safe cleanup candidates.

## Merge Worktree Retention

Merge worktrees should be kept until their merge result is pushed and the final master commit is recorded. Conflict worktrees must not be cleaned automatically.

## Dirty Worktree Skip Rule

Any worktree with non-empty `git status --short` is skipped. Dirty worktrees may contain generated data, conflict files, or uncommitted owner work and require manual review.

## Active Branch Skip Rule

Any worktree with a named checked-out branch is treated as active and skipped. The script only considers detached temporary worktrees that are clean and whose current commit is confirmed reachable from `origin/master`.

## Dry Run

Run dry-run before every cleanup:

```powershell
node scripts/storage-safe-cleanup.mjs --root "C:\Users\47203\Documents" --dry-run
```

Dry-run writes JSON and Markdown reports outside the repository and deletes nothing.

## Apply

Apply only after reviewing the dry-run report:

```powershell
node scripts/storage-safe-cleanup.mjs --root "C:\Users\47203\Documents" --apply
```

Do not use `git clean`, `git reset`, shell `rm`, `del`, or `rmdir` for Dock storage cleanup.
